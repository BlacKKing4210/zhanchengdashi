# 全游戏音乐与音效设计

> 状态：已确认进入实装。目标是让非战斗阶段保持轻松、明亮、欢乐，战斗阶段在同一欢乐主题上增加节奏与推进感，不转为沉重或黑暗的战争音乐。

## 1. 音频目标

- 大厅、编组、抽卡和房间使用同一首日常主题，跨页面不断流、不重新起播。
- 单人和多人战斗使用同主题的加速编曲，保留亲切感，同时加入轻量鼓组、低音和短促推进型旋律。
- 关键操作、养成、抽卡、战斗与结算都有短音效确认。
- 密集战斗不播放连续脚步声，也不高频播放动物叫声；声音只服务于有信息价值的状态变化。
- 音频系统独立于战斗数值、绘制和输入，不从 `_draw_*()` 轮询推断事件。

## 2. 音乐方向

两首循环曲使用同一旋律家族，均为本项目原创的确定性合成音乐，无人声、无第三方素材。

| ID | 页面 / 状态 | 编曲方向 | 规格 |
| --- | --- | --- | --- |
| `menu` | 大厅、编组、抽卡待机、房间 | `104 BPM`，D 大调 / 五声音阶；马林巴、拨弦、温暖铺底、圆润贝斯、沙锤与轻拍手 | 约 `27.7s`，立体声 PCM WAV，无缝循环 |
| `battle` | 单人及房间战斗 | `128 BPM`，D 大调与 Mixolydian 色彩；沿用日常主题，增加军鼓、手鼓、积极低音和短促铜管感 | 约 `22.5s`，立体声 PCM WAV，无缝循环 |

音乐状态规则：

1. 启动游戏后播放 `menu`。
2. 非战斗页面之间切换时不重启音乐。
3. 进入战斗时以 `0.8s` 交叉淡化到 `battle`，并叠加开战短句。
4. 暂停时在 `0.15s` 内降低音乐 `6dB`，继续时恢复，不停止播放头。
5. 结算时降低音乐 `8dB`，叠加胜利、平局或失败短句；返回页面时以 `1.0s` 切回 `menu`。
6. 抽卡与升级只叠加短音效，不切换整首音乐。

## 4. 运行时架构

新增全局 `GameAudio`：

- 两个 `AudioStreamPlayer` 轮换播放音乐并交叉淡化。
- `12` 个短音效播放器组成固定对象池：`10` 个普通战斗声道、`2` 个 UI / 结算保留声道，不在战斗热路径持续创建节点。
- 运行时确保存在 `Music`、`SFX`、`UI` 总线。
- 初始总线音量：`Music -10dB`、`SFX -4dB`、`UI -6dB`。
- `Master` 总线运行时确保装有 `-1dB` ceiling 的 limiter，吸收极端混战的瞬态叠加。
- 音乐与音效可分别启用/禁用，并暴露独立音量接口，为后续设置页保留契约。
- 音乐资源在加载后复制并设置 WAV 正向循环，循环区间覆盖完整采样长度。
- 高频音效使用事件级冷却和轻微音高变化；UI 导航音保持固定音高。
- 战斗世界音效只在镜头附近播放；敌方的生成、攻击与增益不进入全局声场，玩家及友方事件优先。

状态与代码触发点：

- 音乐：`_start_match()`、`_start_multiplayer_match()`、`_return_to_lobby()`、`_finish_battle()`、`_finish_multiplayer_battle()`。
- 页面：`_handle_nav()`、`_handle_room_tap()`、`_show_card_detail()`、`_equip_pending_card_to_slot()`、`_try_upgrade_selected_card()`、`_draw_gacha_rewards()`、`_reveal_next_gacha_card()`。
- 战斗：`_try_unlock()`、`_try_paint_crossed_tile()`、`_spawn_unit()`、`_unit_attack_target()`、`_tower_attack()`、`_damage_unit()`、`_handle_unit_death()`、一次性属性增加函数。
- 暂停：ESC、暂停按钮、继续按钮和退出按钮必须同步音乐 duck 状态。

## 5. 并发、防噪与优先级

| 组 | 最大并发目标 | 全局重触发间隔 |
| --- | ---: | ---: |
| UI | `2` | `40ms` |
| 攻击 / 发射 | `6` | `55ms` |
| 受击 | `6` | `50ms` |
| 生成 | `2` | `100ms` |
| 死亡 | `3` | `90ms` |
| 属性 / 提升 | `2` | `120ms` |
| 解锁 / 经济 | `2` | `180ms` |
| 关键结算 | 不丢弃 | 无普通限流 |

- 全局短音效池固定为 `12` 个播放器；普通池和保留池分别按播放序号替换最早声音，战斗噪声不能占用 UI / 结算保留声道。
- 攻击、受击、死亡使用约 `0.95-1.05` 的轻微随机音高，避免机械重复。
- 同一帧多个群体 Buff 只允许有限数量的 `stat_gain`，不得每个单位同时叠音。
- 玩家直接操作与结算音优先于普通战斗音。

## 6. 资源与生成规范

- 路径：`assets/audio/music/` 与 `assets/audio/sfx/`。
- 音乐：`32kHz`、16-bit、立体声 WAV；短音效：`32kHz`、16-bit、单声道 WAV。
- 峰值不高于约 `-1dBFS`，合成与归一化后不得削波。
- 音乐循环尾音采用环形混音回卷到开头，避免切点爆音和尾音被截断。
- 资源由 `tools/build_game_audio.py` 确定性生成；相同脚本和种子应输出相同音频。
- 当前 PCM WAV 是可直接运行的原创版本；未来如替换为专业母带 OGG，保持事件 ID 与 `GameAudio` 接口不变。

## 7. 验收标准

### P0

- 五个页面状态和两种战斗模式都有正确音乐；非战斗页面切换不重启曲目。
- 所有资源能够被 Godot 4.6 导入与加载，不存在缺失路径或解析错误。
- 战斗开始、暂停、继续、结算和返回页面的音乐状态正确。
- UI、编组、抽卡、房间、解锁、攻击、受击、死亡、成长和结算均有事件反馈。
- 音效触发不改变战斗数值、单位坐标、输入和页面跳转。

### P1

- 音乐循环切点无明显爆音、静音缝隙或音量跳变。
- 理论上限 `72` 单位混战时仍能听清玩家操作与结算提示，不出现持续削波或噪声墙。
- 重复攻击、受击和群体 Buff 受冷却限制；播放器节点数量保持固定。
- 关闭音乐或音效后对应播放器停止，重新启用后可恢复正确状态。

### P2

- 实机 WASAPI 试听确认：日常音乐轻松欢乐；战斗音乐更有推进感，但不沉重、不恐怖、不压迫。
- 大厅与战斗主题能听出旋律亲缘，整体听感属于同一款游戏。
