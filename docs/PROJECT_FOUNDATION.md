# 战城大师项目基础说明

## 目标

本仓库当前完成的是 Godot 游戏开发前期基础工程：先建立稳定的数据、版本控制、文档和自动校验流程，并准备一个可启动的最小 Godot 主场景与配置加载入口。

## 已建立内容

- 通用目录结构：资产、配置表、运行时配置、脚本、场景、测试、工具和文档分区。
- Git 基础环境：`.gitignore`、`.gitattributes`、`.editorconfig`、GitHub 远程地址和主分支约定。
- 配置表系统：CSV 源表、JSON Schema 风格的项目表结构、校验脚本和运行时导出脚本。
- Godot 接入基础：`project.godot`、`scenes/main.tscn`、`ConfigDB` 自动加载脚本。
- GitHub 基础自动化：推送或 Pull Request 时执行配置表校验。
- 项目协作说明：配置表、Git 工作流、工程目录和后续接入建议。

## 开发阶段建议

1. 原型期：先用配置表驱动角色、技能、关卡、掉落和经济数值，避免把设计值写死进代码。
2. 可玩期：确定引擎后建立最小可玩闭环，包括角色移动、战斗、胜负、奖励和配置热更新流程。
3. 内容期：扩大配置表，建立关卡编辑、敌人池、掉落池、局外养成和多语言文本流程。
4. 打磨期：加入性能预算、平台打包、自动化测试、存档兼容和发布分支管理。

## 当前约定

- `config/tables/` 是设计源数据。
- `runtime/config/` 是游戏运行时读取的数据。
- `tools/validate_config.py` 必须通过后再提交配置表变更。
- `tools/export_config.py` 用于同步生成运行时 JSON。
- 大型二进制资产先不启用 Git LFS，等实际资产规模明确后再做迁移。

## 下一步

- 确定目标平台和 Godot 渲染/输入/分辨率基线。
- 在 `scenes/main.tscn` 上建立第一个可运行 Demo 场景。
- 将 `ConfigDB` 接入角色、技能、关卡和掉落读取层。
- 添加最小自动化测试：配置加载、基础战斗公式、关卡生成和掉落结果。
