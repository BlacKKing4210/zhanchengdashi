# 动物通用动作反馈设计

> 状态：效果图评审稿。当前只定义通用动效语言、图片 FX 与 Godot 落地规格；用户明确确认方案并说“实装”前，不修改运行时脚本、场景、资源绑定、输入或点击区域。

## 1. 目标

当前动物以单张 PNG 参与战斗，没有逐动物攻击、受击、移动与死亡动画。本方案不为每个动物制作独立序列帧，而是建立一套所有动物共用的动作反馈：

- 动物原图、身份、白边与卡牌绑定保持不变。
- 动作感由脚点缩放、旋转、绘制位移、闪色和通用图片 FX 组合产生。
- 图片 FX 只表达动作方向、接触、速度、成长和消失，不承担逻辑位置、碰撞或点击。
- 同一套系统覆盖宽体、窄体、长耳、飞行和小体型动物，避免逐动物维护成本。

## 2. 当前工程基线

- 目标画布：`720 x 1280`。
- 经典战斗内场：`592 x 982`，位置 `Rect2(64, 110, 592, 982)`。
- 棋盘：`7 x 13`，共 91 格；`HEX_SIZE = 43`。
- 当前单位图片绘制尺寸：`44 x 44`。
- 当前动物由 `scripts/app/main.gd::_draw_unit()` 即时绘制，不是独立 `Sprite2D` 节点。
- 当前通用 `effects` 只覆盖 `pulse`、`projectile`、`card_popup`，不同事件缺少清楚的动作语义。
- 当前视觉方向仍在审核，故本轮 FX 使用可替换语义色，不绑定此前任何整页效果图。

## 3. 统一动作语法

所有动作都围绕动物脚点执行。逻辑坐标、攻击距离、寻路、地块归属与血条位置保持不变。

| 状态 | 触发与动作拟态 | 建议时长 | 图片反馈 | 复位与中断 |
| --- | --- | ---: | --- | --- |
| 移动 | 轻微上下弹跳、左右倾斜、脚点压缩；方向由实际位移决定 | `0.32s` 循环 | 身后速度线 + 脚边尘点 | 停止移动后 `0.08s` 回正；被攻击/受击/死亡立即打断 |
| 攻击 | `0-0.06s` 反向蓄力压缩，`0.06-0.12s` 朝目标前冲 `8-10px`，随后回弹 | `0.24s` | 朝目标方向的攻击弧 + 接触火花 | 接触帧后进入恢复；死亡最高优先级中断 |
| 受击 | 当帧闪白、横向挤压、沿伤害来源反方向视觉后退 `4-6px` | `0.16s` | 珊瑚红冲击缺口 + 奶白碎片 | 不改变真实坐标；连续受击刷新而不叠加无限位移 |
| 获得属性 | 图片轻弹，语义标记从脚点上浮 `16-20px` | `0.55s` | 攻击/生命/护盾/速度使用同构上升环与不同小图形 | 同类事件 `0.12s` 内合并；持续光环重算不得每帧触发 |
| 获得提升 | 动物 `1.00 -> 1.16 -> 1.00`，一次完整金色确认 | `0.68s` | 金色星环/上升刻度 + 两枚奶白星点 | 只用于卡牌升级确认或未来的重要阶级提升；普通数值增长仍走“获得属性” |
| 死亡 | 保存动物图片快照，短闪后倾倒、缩小、淡出 | `0.42s` | 断裂墨圈 + 尘烟 + 消散碎点 | 死亡快照脱离 `units` 数组；打断并清除该单位其他附着式状态 |

建议顺带补齐出生反馈：脚点 `0.82 -> 1.08 -> 1.00` 弹出并落一圈短尘，`0.25s` 完成，避免与获得提升共用同一金色语义。

## 4. 动效方向候选

三版只比较动作语言与强度，不代表页面视觉方向选择。三版共用同一语义色板，后续可随最终页面风格替换 token。

### A. 弹性墨线动作（推荐）

- 中等粗细墨线、清楚的攻击弧、冲击缺口、速度线、上升箭头、星环和死亡尘圈。
- 形状数量适中，接触帧清楚，在 `44-64px` 动物尺寸下最平衡。
- 推荐作为默认通用套件；Boss 或关键技能未来可在此基础上提高一次强度，不另建语言。

### B. 粗笔战斗印章

- 更粗轮廓、更大的环形缺口与更强的战斗印章感。
- 优点是小屏识别最快；风险是多人同屏时更容易遮挡相邻格与伤害数字。
- 适合作为“强反馈”档，而不建议全量事件都使用。

### C. 克制舞台轨迹

- 更少形状、更细的轨迹、更低的覆盖面积。
- 优点是安静、可长期观看；风险是浅色地块或大量单位同时移动时识别略弱。
- 适合作为减弱动效模式的视觉基线。

## 5. 语义色与形状

本轮 FX 控制在六个可替换 token 内，禁止渐变、外发光与多层阴影：

| Token | 初始色 | 用途 | 非颜色线索 |
| --- | --- | --- | --- |
| `fx_ink` | `#24211D` | 轮廓、死亡断环、速度线 | 断裂/方向形状 |
| `fx_paper` | `#F7E6B5` | 接触闪光、碎片、星点 | 四角星/碎片 |
| `fx_hit` | `#D9543D` | 受击、危险接触 | 锯齿缺口 |
| `fx_power` | `#F2C14E` | 攻击接触、重要提升 | 星环/向上刻度 |
| `fx_gain` | `#2D8C7A` | 属性获得、恢复、护盾 | 上升箭头/环 |
| `fx_motion` | `#4E9BC4` | 移动和速度类属性 | 平行轨迹 |

颜色不能单独承担语义：受击必须有锯齿/碎片，提升必须有星环/上升刻度，死亡必须有断环/消散。

## 6. 图层、锚点与绘制顺序

建议绘制层级：

1. 地块与建筑；
2. 单位背后轨迹、尘点和成长环；
3. 以脚点为轴变换后的动物 PNG；
4. 接触火花、闪白与前景碎片；
5. 血条、队伍标记和伤害数字；
6. 全局 UI。

统一锚点为动物图片底部中心，约等于当前绘制位置的 `pos + Vector2(0, 14)`。图片 FX 以动物世界坐标保存，绘制时再转换到画布坐标；镜头拖动不能写回单位或 FX 世界坐标。

## 7. Godot 落地计划（获批后）

当前架构是单个 `Node2D` 即时绘制，不建议为了本功能把所有动物重构成 `Sprite2D`。

- 使用稳定的 `unit["id"]` 维护 `unit_motion_states`，不能用数组下标，因为死亡过滤会改变下标。
- 附着式状态：`moving`、`move_phase`、`facing`、`attack_time`、`attack_direction`、`hit_time`、`hit_direction`。
- 脱离式效果继续进入 `effects`：`impact`、`stat_gain`、`power_up`、`death_snapshot`、`dust`。
- 主挂点：`_update_units()`、`_unit_attack_target()`、`_damage_unit()`、属性增加函数、`_try_upgrade_selected_card()`、`_handle_unit_death()`、`_spawn_unit()`。
- 优先级：`death_snapshot > hit > attack > move > idle`。
- 运行时 FX 使用一张 atlas 和对象池；不为每个动物复制纹理或创建常驻节点。

## 8. 性能与可访问性

- 同屏活动短效建议上限 `32`，对象池容量建议 `64`；超过上限时先丢弃移动尘点，再合并同类属性反馈。
- 离开战斗可视区的 FX 不绘制；只更新必要计时器。
- 动效只使用绘制变换与透明度，禁止改布局、点击区和逻辑坐标。
- 普通短反馈控制在 `0.15-0.30s`；获得属性与提升允许延长，但不超过 `0.70s`。
- 减弱动效模式关闭旋转、前冲、后退与循环弹跳，仅保留 `0.12-0.25s` 的静态符号、一次轻微缩放和淡出。
- 连续事件必须可中断、可复位，不能锁住输入或让单位停留在错误姿态。

## 9. 评审与验收标准

### P0

- 不修改动物原图、棋盘几何、逻辑位置、碰撞、攻击范围、点击区或战斗数值。
- 六类状态均有独立形状语义；死亡不会因单位当帧移除而消失。
- 任意动画结束或中断后，动物视觉姿态能回到默认状态。

### P1

- 在实际 `44 x 44` 动物尺寸下，玩家可在 3 秒内区分攻击、受击、移动、属性、提升与死亡。
- FX 不遮挡血条、队伍标记、伤害数字或超过一个相邻六边格的主要可读区域。
- 宽体、窄体、长耳、飞行和小体型动物均通过；不出现裁切、边缘穿格或明显洋红残边。
- 同屏多单位、连续受击、攻击与受击并发、属性与死亡打断均保持清楚。

### P2

- 三版 FX 在同一语义色板下仍能看出清楚的强度差异。
- 减弱动效模式不是简单关闭所有反馈，仍保留状态确认。
- 目标分辨率与至少一个窄屏/宽屏边界分辨率截图对照通过。
