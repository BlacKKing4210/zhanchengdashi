# 动物纯程序化动作反馈设计

> 状态：已确认并进入运行时实装。实现原则为“现有动物图片 + 脚点程序化缩放、旋转、绘制位移”，不使用通用图片 FX，不为单个动物制作独立动作帧。

## 1. 目标与边界

当前动物以单张 PNG 参与战斗。所有动物共用同一套姿态算法，用短时形变模拟动作，弥补没有逐动物序列帧的问题。

- 只变换现有动物纹理；不新增 FX 图片、atlas、序列帧、弧线、星环、尘点、碎片或程序化符号。
- 统一脚点为当前 `44 x 44` 图片底边中心：单位画布坐标 `pos + Vector2(0, 14)`。
- 只修改绘制姿态；单位世界坐标、地块、寻路、攻击范围、碰撞、点击区、血条和队伍标记保持不变。
- 动作必须可中断、可刷新、可复位，不阻塞输入和战斗逻辑。

## 2. 六类动作

| 状态 | 程序化姿态 | 时长 | 触发点 |
| --- | --- | ---: | --- |
| 移动 | 按实际位移驱动轻弹，纵向微压缩并随方向左右微倾 | `0.32s` 循环 | `_update_units()` 确实改变 `unit["pos"]` 时 |
| 攻击 | 先反向蓄力压缩，再沿目标方向绘制前冲，最后回弹到脚点 | `0.24s` | `_unit_attack_target()` 结算伤害前 |
| 受击 | 横向挤压，沿远离伤害来源方向绘制后退并轻微倾斜 | `0.16s` | `_damage_unit()` / `_lose_unit_hp()` 实际承受伤害时 |
| 获得属性 | 小幅上提和单次 `1.00 -> 1.10 -> 1.00` 弹起 | `0.36s` | 攻击、生命、护盾、治疗等一次性属性写入点 |
| 获得提升 | 更明显的双段弹起和轻摆，峰值约 `1.18` | `0.55s` | 击杀触发永久成长；牌库卡牌升级成功时使用同语言 |
| 死亡 | 保存原动物纹理快照，向受击方向倾倒、下沉并缩小后消失 | `0.42s` | `_handle_unit_death()`；多人淘汰只生成视觉快照 |

持续光环的每帧重算不得触发属性动作，避免单位持续抖动。出生不是本轮目标，不新增出生动作。

## 3. 状态优先级与并发

优先级为：`死亡 > 受击 > 攻击 > 获得提升 > 获得属性 > 移动 > 待机`。

- 高优先级动作立即打断低优先级动作。
- 低优先级事件可保留一个待播动作，当前动作结束后播放。
- 同类受击刷新计时和方向，不累加位移。
- 移动是无短时动作时的底层循环；停止实际位移后立即回到待机。
- 死亡清除其他状态。死亡单位仍按原逻辑当帧离开 `units`，独立快照只参与绘制与计时，不参与寻敌、单位上限或技能结算。

## 4. 绘制规则

当前单位图片框为 `Rect2(pos + Vector2(-22, -30), Vector2(44, 44))`，脚点是其底边中心。

1. 按原逻辑绘制不变形的阴影。
2. 将现有动物纹理变换原点设为脚点，应用动作返回的 `offset`、`scale`、`rotation`。
3. 在局部坐标 `Rect2(Vector2(-22, -44), Vector2(44, 44))` 绘制动物。
4. 立即恢复画布变换。
5. 按原逻辑绘制不变形的血条和队伍标记。

经典地图与多人地图都保存世界坐标，绘制时才通过 `_world_to_canvas()` 投影。镜头拖动不得写回单位或死亡快照坐标。

## 5. 动作参数

- 移动：垂直偏移不超过 `2.5px`，倾角不超过 `0.045rad`，形变不超过 `4%`。
- 攻击：目标方向绘制前冲不超过 `9px`，倾角不超过 `0.065rad`，形变不超过 `12%`。
- 受击：反来源绘制后退不超过 `6px`，倾角不超过 `0.09rad`，形变不超过 `12%`。
- 属性：上提不超过 `3px`，统一缩放峰值 `1.10`。
- 提升：上提不超过 `5px`，统一缩放峰值 `1.18`，轻摆不超过 `0.05rad`。
- 死亡：倾倒不超过 `0.52rad`，下沉不超过 `8px`，最终缩小到接近零后移除。

所有数值只用于绘制，不反写 `unit["pos"]`、`unit["tile"]` 或其他战斗字段。

## 6. 工程落点

- 短时动作状态放在每个单位字典内，以稳定 `unit["id"]` 隔离，不以数组下标建立外部绑定。
- 纯数学姿态集中到 `scripts/app/systems/unit_motion_feedback.gd`，便于测试并保持 `main.gd` 只负责触发和绘制。
- 死亡快照复用现有 `effects` 生命周期，类型为 `unit_death_snapshot`，只保存 `card_id`、`team`、世界位置、方向、时长与剩余时间。
- 牌库升级成功后，只让详情区现有动物图围绕底边中心弹起和轻摆；升级失败不触发。

## 7. 验收标准

### P0

- 运行时没有新增或引用通用图片 FX、动作 atlas 或逐动物动作帧。
- 六类状态均能从动物本体姿态区分，死亡单位当帧清除后仍能完整播放原图快照动作。
- 攻击、受击、属性、提升和动作计时不改变单位世界坐标、地块、战斗数值或点击区。
- 动作结束、被打断、暂停或重开战斗后，视觉状态能正确复位。

### P1

- `44 x 44` 实际尺寸下动作清楚，但动物不越过一个相邻六边格的主要可读区域。
- 阴影、血条、队伍标记不跟随动物缩放或旋转。
- 连续受击、攻击与受击并发、击杀成长、多人批量淘汰均不串状态。
- 镜头拖动只改变投影后的绘制脚点，不改变单位与死亡快照的世界坐标。

### P2

- 普通短反馈保持在 `0.16-0.36s`，重要提升与死亡不超过 `0.55s`。
- 同屏多单位运行时无持续对象增长、残留姿态或明显绘制污染。
