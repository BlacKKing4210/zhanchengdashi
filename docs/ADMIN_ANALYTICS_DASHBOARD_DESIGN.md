# 竞技数据统计后台设计说明

状态：已实施
任务：OPS-ANALYTICS-001
更新时间：2026-07-18

## 目标

建立一个仅限所有者及被授权运营人员访问的网页后台，以原创的赛事积分榜视觉语言展示：

1. 玩家排行榜、段位、星级、ELO、胜负与场次；
2. 头部玩家当前卡组、卡牌等级和最近一次已记录对局阵容；
3. 所有动物的携带场次、胜场、负场、胜率和样本量；
4. 最近对局及其已冻结的参赛阵容。

界面借鉴国际足球赛事的“积分榜 + 小组排名 + 数据面板”信息层级，不使用 FIFA、世界杯或其他第三方赛事名称、标志、图形资产。

## 范围与边界

### 本次交付

- 新增 dedicated-server 独占的统计投影与持久化；
- 新增独立 Node.js 管理网站，读取脱敏统计快照，而不是原始玩家账号库；
- Owner 与 Analyst 两种后台身份；Owner 可以授权、撤销和审计后台人员；
- 默认仅监听 `127.0.0.1:24568`，并提供 HTTPS 远程部署模式；
- 提供排行榜、卡组、动物胜率、最近对局、授权管理和审计日志页面。

### 非目标与已知限制

- 当前多人战斗仍由房主客户端模拟；本次统计由服务器捕获并持久化首个终局快照，避免重复结算，但不等同于 dedicated-server 战斗模拟。
- 后台会明确显示“服务器记录的房主权威对局”数据来源；要达到反作弊官方战绩，需要后续将战斗模拟和结算迁至 dedicated server。
- 本次不公开普通玩家账号、密码哈希、设备安装 ID、刷新令牌或完整原始资料。

## 页面与 UE

### 赛事总览

- 顶部显示数据更新时间、统计来源和当前管理员；
- 四张 KPI：已记录对局、上榜玩家、过去 24 小时活跃玩家、统计赛季；
- 总览显示最近已完成对局；排行榜、头部卡组与动物统计位于独立页签；
- 移动端按总览、榜单、卡组、动物统计、授权管理的页签顺序显示。

### 玩家榜与卡组

- 以段位、星级、ELO、胜率、场次排序；
- 每名玩家的展示名、用户 ID 和完整账号名均最小化展示：默认显示展示名与截断用户 ID；
- 头部卡组按排行榜前列玩家的最近有效卡组聚合展示。

### 动物胜率

- 定义为“携带该动物的已完成对局胜率”，而非伤害占比或最后一击概率；
- 每行显示动物名称、胜率、胜/负、携带场次及样本量；带有工程/建筑技能标签的动物仍按动物卡统计，真实建筑卡不计入动物胜率；
- 样本为 0 时显示“暂无数据”，不显示误导性的 0%；
- 首版显示全量服务器投影；段位、地图和时间范围筛选属于后续运营筛选能力。

### 授权与审计

- Owner 可新增、禁用、重置后台人员，并查看授权/登录/登出/失败登录审计；
- Analyst 仅可查看数据，不能管理人员或下载包含用户标识的原始数据；
- 删除或禁用为危险操作，必须二次确认并写入审计日志。

## 数据来源与契约

服务器在对局开始时冻结参加者、队伍、卡组和卡牌等级；只有携带正确 `match_id`、覆盖完整且账号唯一的参赛队伍的首个终局快照才会封存统计。队伍赛终局使用固定的 A/B 阵营结果口径，而非房主当前所在阵营；因此随机分位或战斗中房主/权威端迁移都不会反转胜负归因。只纳入满额真人在线房间，不让 AI 补位局或同账号重复占槽局刷入排行榜或动物胜率。服务器启动即登记全量动物目录，因此无样本动物也会以“暂无数据”显示。数据写入：

```text
match_analytics.json (dedicated server 私有数据)
  ├─ matches[match_id]        冻结的参赛阵容、名次/胜负、来源、时间
  ├─ players[user_id]         排行榜投影与最近卡组
  └─ animals[card_id]         携带场次、胜、负与胜率聚合

dashboard_snapshot.json (只读、脱敏、原子写入)
  ├─ overview
  ├─ leaderboard
  ├─ top_decks
  ├─ animals
  └─ recent_matches
```

网站进程只能读取 `dashboard_snapshot.json`，绝不读取 `player_accounts.json`。账号哈希、玩家会话、安装 ID 和刷新令牌不会进入快照或 API 响应。

## 权限和安全设计

| 项目 | 规则 |
|---|---|
| 初始 Owner | 只能在服务器本机通过 CLI 初始化；仓库中没有默认账号或默认密码。 |
| 后台密码 | Node `crypto.scrypt`、独立随机盐、常数时间比较；不复用玩家密码或设备令牌。 |
| 登录保护 | 单 IP 限流、失败审计、固定错误文案、短时会话、登录后轮换会话标识。 |
| Cookie | `HttpOnly`、`SameSite=Strict`；远程 HTTPS 模式强制 `Secure`。 |
| 远程访问 | 默认拒绝非回环绑定；`--allow-remote` 只有同时提供 TLS 证书与私钥时才可启动。 |
| 授权 | Owner 可创建/禁用 Owner 或 Analyst；禁用立即使其会话失效。 |
| 写操作 | 带 CSRF 令牌、同源检查和审计日志；只读 API 不返回高敏感字段。 |

## 视觉与交互规范

- 桌面优先的数据密度布局：深靛蓝积分板、暖金强调、白色数据面；
- 使用文本、色彩和清晰标签传达统计状态，不依赖单独颜色表达胜负；
- 表格保留横向滚动以适配窄屏；首版按服务器快照排序，不提供浏览器端排序；
- 颜色不单独表达胜负：绿色/红色旁始终配合“胜/负”文字和图标；
- 所有表格在小屏幕降级为卡片列表，筛选条件保留在 URL；
- 状态变化仅使用 150-250ms 的 opacity/transform 动效，并遵守 `prefers-reduced-motion`。

## 验收标准

1. 未初始化 Owner 时，网站无可登录默认账号；首次 Owner 只能由服务器本机 CLI 创建。
2. 普通玩家账号、玩家 ENet 会话或设备令牌不能访问网站 API。
3. 未登录访问任何数据页/API 返回登录页或 `401`；Analyst 无法执行人员管理操作。
4. 一个 `match_id` 重复接收终局快照，只产生一条对局和一次动物统计。
5. 每个动物胜率都带样本量；没有样本时显示“暂无数据”。
6. 排行榜与头部卡组来自服务器写出的脱敏快照，而不是浏览器或原始账号库。
7. 默认仅监听回环地址；远程监听缺少 TLS 证书/私钥时明确启动失败。
8. 键盘可登录和退出；核心文本和控件达到 WCAG AA 对比度。

## 验证计划

- Godot：统计存储单元测试、房间 roster 冻结、重复终局快照去重测试；
- Node：Owner 初始化、登录限流、Owner 邀请/禁用、Analyst 越权、Cookie/CSRF、快照读取和动物胜率渲染测试；
- 浏览器：127.0.0.1 冒烟、未授权拦截、桌面与窄屏截图检查；
- 运维：本机绑定、HTTPS 远程绑定保护、停止/重启后统计与授权数据保持。
