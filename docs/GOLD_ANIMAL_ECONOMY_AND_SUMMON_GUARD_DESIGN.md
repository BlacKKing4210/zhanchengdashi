# 产金币动物数值与死亡召唤保护

更新日期：2026-07-18

## 目标

- 将所有能为己方产金币的动物改为结构化、可配置且与游戏内说明一致的技能。
- 根据触发频率，将单次金币数提高为旧值的 5--10 倍；持续/高频触发使用较低倍率，死亡和占领等低频条件使用较高倍率。
- 保留正常单位出生和非循环死亡召唤的技能；仅阻断死亡召唤谱系中的循环后代继续触发技能。

## 数值原则

| 触发类别 | 目标倍率 | 原因 |
| --- | ---: | --- |
| 攻击后概率、周期、友军阵亡 | 5 倍 | 一局可能反复触发，收益需受频率约束。 |
| 阵亡 | 8 倍 | 一次性且需要牺牲单位。 |
| 参与占领 | 8 倍 | 需要推进并完成地图条件，触发较少。 |

金币技能必须由 `skill_trigger`、`skill_effect`、`skill_power` 和可选 `skill_chance` 驱动；显示文案从相同数据生成，不能再依赖与实际逻辑不同的自然语言关键词。

| 动物 | 触发 | 调整后收益 | 倍率 |
| --- | --- | ---: | ---: |
| 鸡 | 自身阵亡 | +8 | 8 倍 |
| 鸽子 | 自身阵亡 | +24 | 8 倍 |
| 仓鼠 | 每 8 秒 | +10 | 5 倍 |
| 狗 | 攻击后，35% 概率 | +5 | 5 倍 |
| 松鼠 | 参与占领 | +8 | 8 倍 |
| 猪 | 自身阵亡 | +40（己方） | 8 倍 |
| 水獭 | 每个友军阵亡 | +10 | 5 倍 |
| 牛 | 参与占领 | +10 | 10 倍 |

## 召唤循环保护

每个由死亡技能生成的单位保留 `death_summon_lineage`。生成前把死亡单位的卡牌 ID 写入谱系；若待生成卡牌已在谱系中，新单位保留战斗实体和召唤反馈，但设置 `skill_triggers_enabled = false`。

因此：

- 首只鸭死亡仍生成 1 只鸭。
- 这只替补鸭不能再触发自身死亡召唤，因此死亡后不会产生第三只鸭。
- 银背大猩猩死亡生成大猩猩不构成循环，生成单位仍能正常使用技能。
- 未来 A -> B -> A 的间接死亡召唤循环也会在闭环处停止。

死亡动画、单位移除和其他正常单位对“友军死亡”的响应保持有效；被禁用的仅是循环后代自身的技能入口。

## 验收

1. 所有金币动物的单次收益、触发时机与卡牌说明一致，且数据导出到 `runtime/config/cards.json`。
2. 普通鸭死亡后恰好生成一只带召唤反馈的替补鸭；替补鸭死亡后不再新增单位。
3. 非循环死亡召唤仍正常可用。
4. 通过配置校验、GDScript 缩进检查、动物技能和金币技能回归测试。
