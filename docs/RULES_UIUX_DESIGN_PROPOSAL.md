# 战城大师规则与 UI/UE 设计案

版本：v0.2 提案

生成日期：2026-07-01

## 1. 设计案目标

这份文档不是当前实现说明，而是一份下一版可落地的规则与 UI/UE 设计案。目标是把当前原型从“能跑的地块战斗”推进到“规则清楚、选择有意义、界面能指导玩家”的版本。

本版本优先解决四个问题：

- 编组要真实影响战斗，而不只是展示卡牌。
- 地块购买要形成清楚的风险和收益，而不是单纯点亮地图。
- 经济、战斗、胜负要能支撑 2-3 分钟一局的节奏。
- UI 要让玩家知道下一步该做什么、为什么买、买了会得到什么。

## 2. 产品体验定位

一句话定位：

玩家用 8 张出战卡组成军团，在 7x13 六边形战场上购买相邻地块，翻出单位、防御和经济建筑，靠自动推进的部队摧毁敌方基地或在限时内赢得领土优势。

目标体验：

| 维度 | 设计目标 |
| --- | --- |
| 单局长度 | 2-3 分钟 |
| 每局关键选择 | 12-18 次地块购买，3-6 次高价值风险选择 |
| 操作复杂度 | 单指点击为主，少量长按查看详情 |
| 战斗理解 | 玩家不微操单位，重点看地图连接、建筑生产和资源投入 |
| 成长驱动 | 局外编组、卡牌收集、卡牌升级、关卡奖励 |
| 视觉气质 | 饱和、厚描边、清晰反馈、移动端大按钮 |

## 3. 核心循环

### 3.1 局外循环

```text
选择关卡 -> 调整编组 -> 开始战斗 -> 获得奖励 -> 强化卡牌/调整编组 -> 挑战下一关
```

局外要让玩家做两类事：

1. 战前准备：看推荐战力、敌人倾向、地块规则，选择一套编组。
2. 战后成长：获得金币、经验、卡牌碎片或道具，用于升级卡牌和解锁新关卡。

### 3.2 局内循环

```text
观察地图 -> 选择相邻地块 -> 判断价格和收益 -> 购买/等待收入 -> 翻开建筑或奖励 -> 单位自动推进 -> 扩张更多地块
```

局内要让玩家持续回答三个问题：

- 现在买便宜地块铺开，还是攒钱买高价值地块？
- 先扩经济，还是先补单位和防御？
- 当前战线危险时，要买塔防守，还是买单位反推？

## 4. v0.2 推荐规则总览

| 模块 | v0.2 推荐值 |
| --- | --- |
| 地图 | 7 列 x 13 行六边形地图 |
| 战斗时长 | 180 秒 |
| 玩家初始金币 | 60 |
| 敌方初始金币 | 60，AI 使用相同经济规则 |
| 基地收入 | 每 3 秒 +12 金币 |
| 金矿收入 | 每 3 秒 +10 金币 |
| 问号地块价格 | 25 |
| 金矿价格 | 50 |
| 单位/防御地块价格 | 开局随机 50 / 100 / 250 |
| 编组数量 | 5 套 |
| 每套编组 | 1 英雄位 + 7 单位位 |
| 当前版本开放 | 全部卡牌开放，后续接抽卡/关卡解锁 |
| 胜利条件 | 摧毁敌方基地，或倒计时结束后分数领先 |

## 5. 编组规则

### 5.1 编组结构

当前 UI 已有 8 个出战位。v0.2 建议定义为：

| 槽位 | 类型 | 作用 |
| --- | --- | --- |
| 1 | 英雄位 | 提供全局被动、基地技能或战斗开局加成 |
| 2-8 | 单位位 | 决定本局单位地块可翻出的单位池 |

如果玩家把非英雄卡放入 1 号位，系统应提示“英雄位需要英雄卡”。当前版本如果暂不做限制，可以先允许任意卡，但 UI 上预留英雄位标识。

### 5.2 编组影响战斗

v0.2 的关键变化：单位地块不再从全局单位池抽，而是从玩家当前编组的 7 张单位卡中抽。

单位地块翻开逻辑：

1. 战斗开始时读取当前编组。
2. 单位地块根据价格档位生成品质倾向。
3. 在当前编组中筛选符合品质倾向的单位。
4. 如果没有符合单位，则向下兼容到低一档品质。
5. 最终抽出 1 张单位卡，生成对应兵营或大厅。

推荐品质倾向：

| 地块价格 | 首选品质 | 次选品质 | 设计意图 |
| ---: | --- | --- | --- |
| 50 | common | rare | 早期铺场，稳定但不爆炸 |
| 100 | rare | common / epic | 中期主力选择 |
| 250 | epic / legendary | rare | 高投入高回报 |

### 5.3 英雄位作用

v0.2 英雄先不作为可移动单位上场，避免复杂操作。建议做成被动加成：

| 英雄 | 推荐能力 | UI 表现 |
| --- | --- | --- |
| 守城先锋 | 己方基地和防御塔 HP +10% | 编组页显示盾牌图标 |
| 游击弓手 | 己方远程单位攻击范围 +8% | 编组页显示箭矢图标 |

后续可以把英雄升级为可释放主动技能，例如“基地炮击”“全军冲锋”“防线护盾”。

### 5.4 卡牌基础信息

每张卡至少显示：

- 名称
- 品质
- 等级
- 费用/权重
- HP
- 攻击
- 射程
- 标签
- 技能摘要

卡牌详情弹窗显示完整信息，编组卡面只显示最关键的 3-4 项。

## 6. 地图与地块规则

### 6.1 地图生成

v0.2 使用配置表目标规则：

- 地图为 7x13。
- 玩家基地固定在下方中线。
- 敌方基地固定在上方中线。
- 开局在上半图随机地块类型和价格。
- 再按水平轴镜像到下半图，保证双方结构公平。

### 6.2 地块状态

每个地块有 5 种状态：

| 状态 | 说明 | UI 表现 |
| --- | --- | --- |
| 未发现 | 不显示具体类型 | 暗色六边形，弱轮廓 |
| 可购买 | 与己方连接且金币足够 | 高亮边框，显示价格 |
| 金币不足 | 与己方连接但金币不足 | 灰色价格，点击提示差额 |
| 已占领 | 属于己方或敌方 | 使用阵营颜色填充 |
| 交战中 | 双方单位或占领进度变化 | 进度环和战斗特效 |

### 6.3 地块类型

| 类型 | 出现概率 | 价格 | 翻开结果 |
| --- | ---: | --- | --- |
| 问号 | 50% | 25 | 空、防御、单位、金币 |
| 单位 | 20% | 50/100/250 | 从当前编组抽单位 |
| 防御 | 20% | 50/100/250 | 从防御池抽塔 |
| 金矿 | 10% | 50 | 建造金矿 |
| 基地 | 固定 | 0 | 开局固定存在 |

### 6.4 问号地块

问号地块是低价低确定性的节奏工具。

| 结果 | 概率 | 说明 |
| --- | ---: | --- |
| 空地 | 70% | 只获得占领地块和连接价值 |
| 基础单位 | 10% | 从当前编组低价单位倾向抽 1 张 |
| 基础防御 | 10% | 抽 1 个普通/稀有防御 |
| 金币 | 10% | 立即获得 25-40 金币 |

体验目标：

- 玩家不会因为问号必赚而无脑点。
- 问号仍然值得用于连接和低价探索。
- 问号翻出奖励时有明显惊喜反馈。

### 6.5 连接与购买

购买规则：

1. 只能购买与己方已占领地块相邻的地块。
2. 不能购买敌方已占领地块。
3. 不能购买已有建筑的地块。
4. 金币不足时不能购买，但要显示还差多少金币。
5. 购买成功后立即翻开并播放卡牌揭示或建筑落地反馈。

### 6.6 占领规则

推荐占领公式：

```text
占领速度 = 基础速度 0.35/s + 己方单位数量 * 0.25/s
争夺状态 = 双方都有单位时，占领暂停并缓慢回退
```

细则：

- 空地可被单位占领。
- 有建筑的地块必须先摧毁建筑，才能开始占领。
- 占领完成后地块归属改变。
- 占领进度用六边形外圈显示。

## 7. 经济规则

### 7.1 推荐经济节奏

| 来源 | 金币 | 周期 | 说明 |
| --- | ---: | ---: | --- |
| 初始金币 | 60 | 开局一次 | 足够买 2 个问号或 1 个金矿后等收入 |
| 基地 | 12 | 3 秒 | 保底收入 |
| 金矿 | 10 | 3 秒 | 扩经济的核心收益 |
| 问号金币 | 25-40 | 立即 | 小惊喜，不能稳定依赖 |
| 摧毁敌方建筑 | 8-20 | 立即 | 鼓励进攻，可后续接入 |

### 7.2 经济目标曲线

| 时间段 | 玩家状态 | 经济目标 |
| --- | --- | --- |
| 0-30 秒 | 开局探索 | 买 2-4 个低价地块，至少发现 1 个生产点 |
| 30-90 秒 | 扩张成型 | 形成 2-4 个单位/防御建筑，争夺中线 |
| 90-150 秒 | 压力升级 | 高价地块开始影响局势 |
| 150-180 秒 | 终局冲刺 | 摧毁基地或抢分 |

### 7.3 价格设计

| 价格 | 定位 | 玩家心理 |
| ---: | --- | --- |
| 25 | 探路和连接 | “可以马上试一下” |
| 50 | 基础投资 | “稳定买一个功能点” |
| 100 | 中期选择 | “我要为这个方向负责” |
| 250 | 高风险核心 | “攒钱赌一波强卡或强塔” |

## 8. 建筑规则

### 8.1 建筑定位

| 建筑 | 作用 | 推荐 HP | 推荐机制 |
| --- | --- | ---: | --- |
| 基地 | 核心胜负点 | 520 | 攻击附近敌人，产出金币 |
| 兵营 | 生产普通/稀有单位 | 170 | 周期生成所抽单位 |
| 大厅 | 生产史诗/传说单位 | 210 | 周期更慢，单位更强 |
| 防御塔 | 守线和清敌 | 180-300 | 根据塔类型攻击 |
| 金矿 | 经济增长 | 125 | 每 3 秒产出金币 |
| 空地 | 连接和计分 | 0 | 无建筑 |

### 8.2 建筑生产

推荐生产周期：

| 单位品质 | 建筑 | 生产周期 |
| --- | --- | ---: |
| common | 兵营 | 3.5 秒 |
| rare | 兵营 | 4.5 秒 |
| epic | 大厅 | 5.8 秒 |
| legendary | 大厅 | 7.2 秒 |

### 8.3 防御塔类型

| 防御 | 定位 | 推荐表现 |
| --- | --- | --- |
| 哨戒箭塔 | 稳定单点 | 射击最近敌人 |
| 火炮塔 | 范围伤害 | 炮弹落点小范围爆炸 |
| 修复信标 | 防守续航 | 周期治疗附近友方建筑/单位 |
| 风暴方尖碑 | 高价清群 | 链式闪电打多个目标 |

## 9. 单位与战斗规则

### 9.1 单位字段

战斗应读取 `units.csv` 的这些字段：

| 字段 | 战斗用途 |
| --- | --- |
| max_hp | 最大生命 |
| move_speed | 移动速度 |
| attack_range | 攻击范围 |
| base_damage | 基础伤害 |
| attack_cooldown_sec | 攻击冷却 |
| skill_id | 技能 |
| tags | AI 和效果标签 |

### 9.2 单位行为

单位自动行动，不需要玩家微操。

优先级：

1. 如果攻击范围内有敌方单位，攻击最近敌方单位。
2. 如果当前地块有敌方建筑，攻击建筑。
3. 如果附近有可占领地块，推进占领。
4. 否则向敌方基地路径移动。

### 9.3 技能接入阶段

v0.2 不建议一次接完整复杂技能。推荐分三档实现：

| 阶段 | 技能类型 | 示例 |
| --- | --- | --- |
| 第一阶段 | 伤害、治疗、护盾、减速 | 最容易读懂和调参 |
| 第二阶段 | 眩晕、弹射、范围、击退 | 增加战术差异 |
| 第三阶段 | 召唤、复活、光环、风险换血 | 做阵容核心 |

### 9.4 单位上限

推荐加入单位上限，避免性能和画面失控：

| 项 | 推荐值 |
| --- | ---: |
| 全场单位上限 | 80 |
| 单阵营单位上限 | 40 |
| 单建筑同屏单位软上限 | 6 |

超过软上限时，该建筑暂停生产，直到附近单位减少。

## 10. 敌方 AI 规则

### 10.1 AI 目标

敌方 AI 不需要很聪明，但要像一个稳定对手。

AI 目标优先级：

1. 保证基地周围有连接和防御。
2. 优先购买便宜相邻地块扩张。
3. 中期争夺中线。
4. 如果玩家单位压力大，优先买防御。
5. 如果敌方金币充足，尝试购买高价单位/防御地块。

### 10.2 AI 行动频率

| 阶段 | 时间 | 行动间隔 |
| --- | ---: | ---: |
| 开局 | 0-30 秒 | 1.6-2.1 秒 |
| 中期 | 30-120 秒 | 1.1-1.6 秒 |
| 终局 | 120-180 秒 | 0.8-1.2 秒 |

### 10.3 AI 购买评分

推荐评分：

```text
score =
  可连接价值
  + 地块类型价值
  + 战线距离价值
  + 当前局势修正
  - 价格压力
```

地块类型基础价值：

| 类型 | 分值 |
| --- | ---: |
| 金矿 | 80 |
| 单位 | 70 |
| 防御 | 65 |
| 问号 | 40 |

局势修正：

- 敌方单位接近基地时，防御 +40。
- 金币收入落后时，金矿 +35。
- 中线地块，单位 +25。
- 高价地块在金币不足 300 时，分值 -30。

## 11. 胜负与结算

### 11.1 胜利条件

| 条件 | 结果 |
| --- | --- |
| 摧毁敌方基地 | 立即胜利 |
| 己方基地被摧毁 | 立即失败 |
| 时间结束 | 按分数结算 |

### 11.2 计分公式

推荐公式：

```text
最终分数 =
  占领地块数 * 2
  + 建筑数量 * 5
  + 金矿数量 * 4
  + 敌方基地损血百分比 * 20
  - 己方基地损血百分比 * 10
```

这样能避免“只铺地不进攻”过强，同时让打残敌方基地有意义。

### 11.3 结算展示

结算页显示：

- 胜利/失败
- 领土数
- 建筑数
- 摧毁建筑数
- 基地剩余 HP
- 获得奖励
- 继续 / 再来一次 / 调整编组

## 12. UI 信息架构

### 12.1 底部导航

底部 5 项保持现有结构：

| 位置 | 名称 | v0.2 状态 |
| --- | --- | --- |
| 左 1 | 商店 | 锁定展示，显示“即将开放” |
| 左 2 | 编组 | 开放 |
| 中间 | 战斗 | 开放，返回大厅/战斗入口 |
| 右 2 | 抽卡 | 锁定展示，显示“即将开放” |
| 右 1 | 更多 | 锁定展示，可先接设置/公告 |

中间“战斗”按钮应永远最突出。其他锁定入口要可见，但不能抢 CTA。

### 12.2 大厅页面

大厅目标：让玩家一眼知道“当前上阵阵容”和“点哪里开始”。

必备信息：

- 顶部资源条：金币、宝石、能量、卡券。
- 中央装饰场景：当前阵营代表角色/基地。
- 当前关卡卡片：关卡名、推荐战力、奖励预览。
- 当前编组预览：显示英雄和 4-5 张核心卡。
- 开始战斗按钮：最大、黄色/橙色、底部上方。
- 底部导航：5 项。

大厅布局草图：

```text
[资源条/玩家信息]

        [当前关卡名 / 推荐战力]
        [基地和单位装饰场景]

     [当前编组小卡预览 1 2 3 4 5]

          [开始战斗 CTA]

[商店] [编组] [战斗] [抽卡] [更多]
```

### 12.3 编组页面

编组目标：让玩家快速替换出战卡，理解阵容强弱和缺口。

区域：

1. 顶部页签：编组 1-5。
2. 出战区：1 英雄位 + 7 单位位。
3. 阵容摘要：战力、平均费用、近战/远程/坦克/支援数量。
4. 卡牌列表：全部卡牌，支持筛选和排序。
5. 详情侧栏/弹窗：点卡牌后显示完整信息。

编组布局草图：

```text
[编组] [1] [2] [3] [4] [5]

[英雄位] [单位] [单位] [单位]
[单位]   [单位] [单位] [单位]

[战力] [平均费] [近战] [远程] [支援]

[筛选: 全部 品质 职能 族群] [排序]

[所有卡牌网格...]

[底部导航]
```

交互：

- 点击出战位：选中槽位，黄框高亮。
- 点击卡牌：放入选中槽位。
- 点击已上阵卡：交换位置。
- 长按卡牌：打开详情。
- 点击筛选：展开小菜单。
- 编组不合法时：开始战斗按钮置灰，并提示原因。

### 12.4 卡牌详情弹窗

详情弹窗信息：

| 区域 | 内容 |
| --- | --- |
| 顶部 | 卡牌头像、名称、品质、等级 |
| 中部 | HP、攻击、射程、攻速、费用 |
| 标签 | 近战、远程、坦克、支援、控制等 |
| 技能 | 技能名、冷却、效果 |
| 操作 | 上阵/替换/关闭 |

详情弹窗不要挡住底部导航以外的关键区域；移动端建议底部抽屉式弹出。

### 12.5 战斗页面 HUD

战斗目标：不用文字教学，也能看懂“钱、时间、当前可买什么、战线在哪里”。

顶部：

- 玩家头像和基地 HP。
- 当前金币。
- 倒计时。
- 敌方基地 HP。

地图：

- 六边形地块居中。
- 可购买地块发亮。
- 价格显示在地块上。
- 占领进度用外圈。
- 建筑生产进度用小进度条。

底部选中面板：

- 未选中：显示简短目标，例如“选择相邻地块扩张”。
- 选中可买地块：显示类型、价格、可能产出、购买按钮。
- 选中建筑：显示 HP、产出、下一次生产时间。
- 金币不足：购买按钮变灰，显示还差多少。

战斗布局草图：

```text
[我方基地HP] [金币]      [倒计时]      [敌方基地HP]

             [六边形战场]
    [单位自动推进 / 建筑 / 占领进度]

[选中地块信息]                 [购买/查看按钮]
```

### 12.6 结算页

结算页目标：让玩家明白为什么赢/输，并给下一步。

展示顺序：

1. 胜利/失败标题。
2. 关键数据：领土、建筑、基地 HP、摧毁数。
3. 奖励：金币、卡牌碎片、道具。
4. CTA：继续。
5. 次要按钮：再来一次、调整编组。

## 13. UI 视觉规范

### 13.1 风格

使用当前 brawler arcade UI 方向：

- 厚深色描边。
- 硬阴影。
- 高饱和蓝紫底。
- 黄色/橙色主按钮。
- 红色警告，绿色可领取/成功。
- 卡牌品质使用绿色、蓝色、紫色、金色。
- 所有按钮保证 48px 以上触控高度。

## 14. 新手引导与 UE

### 14.1 第一局引导

第一局只教三件事：

1. 点相邻地块购买。
2. 金矿能提高收入。
3. 摧毁敌方基地或抢更多地块能胜利。

不要在第一局解释完整卡池、品质、技能和抽卡。

### 14.2 引导触发

| 触发 | 提示 |
| --- | --- |
| 第一次进大厅 | “先检查编组，也可以直接开始战斗” |
| 第一次进编组 | “点击出战位，再选择卡牌替换” |
| 第一次战斗 | “购买发光的相邻地块扩张” |
| 金币不足 | “等待收入或优先占领金矿” |
| 第一次买金矿 | “金矿会持续产出金币” |
| 第一次接近失败 | “防御塔可以守住基地附近地块” |

### 14.3 错误恢复

UI 要帮玩家从错误中恢复：

- 买不起时显示还差多少，而不是只说金币不足。
- 点不可购买地块时，高亮最近可购买地块。
- 编组不合法时，直接标出问题槽位。
- 战斗失败后推荐“调整编组”而不是只给再来一次。

## 16. 数据配置落地建议

### 16.1 需要新增或调整的表

| 表 | 用途 |
| --- | --- |
| decks.csv | 保存默认编组、槽位规则、英雄位限制 |
| card_levels.csv | 卡牌等级、碎片、升级消耗、数值成长 |
| battle_rules.csv | 战斗时长、初始金币、计分公式、单位上限 |
| building_rules.csv | 建筑 HP、生产周期、攻击参数 |
| ai_rules.csv | AI 行动频率、购买评分权重 |
| ui_text.csv | UI 固定文案和提示 |

### 16.2 优先配置化顺序

1. `battle_rules.csv`：先把战斗时长、初始金币、收入周期放进去。
2. `building_rules.csv`：把建筑 HP、攻击、生产周期迁出 `main.gd`。
3. `decks.csv`：让编组槽位和英雄位规则数据化。
4. `ai_rules.csv`：让敌方购买节奏可调。
5. `card_levels.csv`：后续接升级和碎片。

## 17. 实现拆分建议

### 17.1 第一阶段：规则打通

目标：编组能影响战斗。

任务：

- 战斗开始读取当前编组。
- 单位地块从当前编组抽单位。
- 生成建筑时绑定单位 ID。
- 单位生成时读取 `units.csv` 数值。
- 战斗 HUD 显示真实卡牌名。

验收：

- 换编组后，战斗中出现的单位会变化。
- 同一地块价格能影响抽出单位品质。

### 17.2 第二阶段：UI 完整化

目标：玩家清楚每次购买的代价和可能收益。

任务：

- 战斗选中面板完善。
- 地块价格和类型图标完善。
- 卡牌详情弹窗。
- 编组筛选和排序。
- 结算数据面板。

验收：

- 玩家不看文档也能完成一局。
- 金币不足、不可购买、购买成功都有明确反馈。

### 17.3 第三阶段：成长和开放入口

目标：局外循环成立。

任务：

- 关卡选择。
- 奖励结算。
- 卡牌碎片。
- 卡牌升级。
- 商店或抽卡至少开放一个。

验收：

- 完成一局后有可用奖励。
- 奖励能改变下一局编组或战力。

## 18. v0.2 验收清单

规则验收：

- 当前编组会影响单位地块产出。
- 单位地块价格会影响品质倾向。
- 金矿和基地按同一收入周期结算。
- 敌方 AI 使用相同基础经济。
- 倒计时结算分数可解释。

UI/UE 验收：

- 大厅 1 秒内能看到开始战斗按钮。
- 编组页能在 3 次点击内替换一张卡。
- 战斗页能一眼分辨可购买、金币不足、敌方地块。
- 购买成功、翻牌、建筑生成、金币不足都有反馈。
- 结算页能解释胜负和奖励。

技术验收：

- 修改配置表后 `tools/validate_config.py` 通过。
- 运行时 JSON 与 CSV 同步。
- 战斗能连续跑 5 局无崩溃。
- 全场单位达到上限时不会卡顿或刷爆。
