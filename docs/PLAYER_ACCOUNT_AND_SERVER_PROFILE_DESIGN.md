# 玩家账号与服务器资料设计

版本：v1.1（2026-07-14）

## 默认设备账号

- 客户端首次启动时生成 32 字节随机“安装 ID”，保存在 `user://client/device_account.json`，不读取 MAC、硬盘序列号或其他物理地址。
- 首次连接服务器时，客户端用安装 ID 发起设备认证。服务器创建游客账号、生成不可修改的 `user_id`，并仅在这次响应中返回 32 字节随机长期登录令牌。
- 客户端保存长期令牌；后续连接默认用同一安装 ID 与长期令牌自动登录，无需玩家重复输入账号密码。
- 服务器只保存安装 ID 的 SHA-256 摘要、随机盐和长期令牌哈希，不保存安装 ID或令牌明文。网络连接仍使用短期会话令牌访问玩家资料。
- “注销账号”只结束当前服务器会话，不删除本机长期令牌；下次启动或重新连接时仍默认进入原账号。手动账号与密码功能作为兼容和未来绑定入口保留。
- 删除应用数据或丢失本机凭据会失去自动登录能力；正式发行前应补充账号绑定、令牌轮换和跨设备找回流程。

## 目标

每名玩家拥有服务器生成且不可修改的 `user_id`、唯一账号和密码凭据。密码只保存加盐哈希。卡牌拥有数量、卡牌星级、出战编组、抽卡券、段位星数与 ELO 均由服务器资料作为权威数据。

## 模块边界

- `PlayerAccountStore`：仅运行在服务端，负责账号注册、密码校验、会话令牌和 JSON 持久化。
- `OnlineRoom`：负责账号 RPC、把网络连接映射到已登录用户，以及限制资料写入只能作用于当前会话用户。
- `Main`：展示账号中心、发起注销、调整本机音量；不保存密码，不把客户端资料当作服务器权威数据。
- 账号数据库默认位于服务端 `user://server/player_accounts.json`，可用测试路径覆盖。

## 数据结构

账号记录包含 `user_id`、规范化账号、密码盐、密码哈希、创建/更新时间和 `profile`。资料字段：

- `card_counts`：卡牌 ID 到拥有数量。
- `card_levels`：卡牌 ID 到星级，最小 1。
- `deck`：最多 8 个卡牌 ID。
- `gacha_tickets`：非负整数。
- `rank_stars`、`elo`：非负整数。

## 页面与跳转

主界面点击中央基地 -> 账号中心；基地保持可点击，但主界面不额外显示“点击基地查看账号”等教学说明。未登录时账号中心提供账号、密码输入以及注册/登录按钮；登录成功后显示服务器生成的 UserID。关闭 -> 主界面；注销 -> 清除会话并停留在账号中心；玩家协议 -> 协议文本层；协议返回 -> 账号中心。音乐与音效按钮即时切换并显示当前状态。

账号中心显示 UserID 和服务器同步状态。未登录时显示“未登录”，客户端不得伪造 UserID 或宣称资料已同步。

## 安全与失败处理

- 账号比较不区分大小写，长度 3-32；密码长度 8-72。
- 密码使用随机盐和多轮 SHA-256 派生；数据库不保存明文密码和会话令牌。
- 登录失败使用统一错误，不暴露账号是否存在。
- 资料保存采用临时文件替换；字段在服务端归一化，负数和未知类型被拒绝或修正。
- 登录后客户端先应用服务器资料，再以变更检测方式同步卡牌数量、卡牌星级、编组、抽卡券和段位资料；未登录状态不上传。
- 断线与注销都会解除 peer 到用户的映射。

## 验收

- 注册产生唯一 UserID，重复账号被拒绝，正确/错误密码可区分成功与失败。
- 重启存储模块后资料仍可读取，卡牌、星级、编组、抽卡券与段位均保持。
- 未登录连接不能读写资料；已登录连接只能读写自己的资料。
- 点击主页基地可打开账号中心；UserID、注销、协议、音乐和音效控件均有真实行为。
- 未登录玩家可在账号中心注册或登录；登录成功会加载服务器资料，后续游戏数据变化会自动回存。
