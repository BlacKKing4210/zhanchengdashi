# 新版召唤动物与流派设计案

生成日期：2026-07-08

用途：本文由 `config/tables/cards.csv` 反向整理当前 60 张动物卡的品质、属性、速度、射程和 `skill_text` 定位。`cards.csv` 是权威来源；本文不再作为上游数值约束，也不要求把表格改回旧流派案。

## 1. 设计口径

- 先认表，再归纳：当前动物定位以 `cards.csv` 已配置内容为准，策划案负责解释这些配置在卡池里的角色。
- 表格改动后立即运行 `tools/validate_config.py` 和 `tools/export_config.py`，游戏读取 `runtime/config/cards.json`。
- `skill_text` 有内容时，它同时承担卡面说明、玩家可读定位和当前战斗原型的关键词落地依据。
- 如果未来某条技能需要精确概率、范围、目标过滤或数值公式，应在保持 `skill_text` 可读的同时增加结构化字段；不要让旧文档覆盖新表。

## 4. 技能落地口径

当前战斗原型已经按 `skill_text` 识别以下关键词族：

- 召唤数量：`数量+1`、`数量+2`、`额外召唤一个...`。
- 目标偏好：优先建筑、远程、后排、前排。
- 经济：阵亡给金币、攻击概率给金币、参与占领给金币、友军阵亡给金币、对方获得金币。
- 成长/增益：随机友军攻击或生命、所有友军生命、我方/所有动物攻击、速度光环、击杀成长、受击成长。
- 战斗效果：先手攻击、反伤、受到伤害 -1、攻击翻倍、额外 50% 伤害、360° AOE、额外攻击目标、剧毒减速、攻击/受击自损、附近友军承伤、阵亡召唤。

注意：现在是轻量关键词落地，适合自动战斗原型验证。未来若要让“50%概率”“范围”“附近”“最大生命值”等全部精确可调，应把这些参数拆到新的结构化字段，同时继续保留 `skill_text` 做玩家可读描述。

## 6. 后续调优原则

- 先改表：动物品质、攻击、生命、速度、射程、召唤间隔和 `skill_text` 都先改 `config/tables/cards.csv`。
- 再导出：每次改 CSV 后立刻校验并导出，保证 `runtime/config/cards.json` 与 CSV 一致。
- 再验证：进入战斗检查卡面显示、地块详情、召唤速度、攻击范围、目标偏好和技能触发。
- 再决定是否结构化：如果某句 `skill_text` 已经证明好玩但需要更精确的概率/范围/数值，再增加专门配置字段。
