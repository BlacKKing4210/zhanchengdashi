# 动物技能实装验收

更新日期：2026-07-17

## 产品负责人结论

当前动物技能不能判定为“全部已按描述实装”。卡牌表中同时存在结构化技能字段与旧版自然语言描述，部分卡牌的两者含义冲突；运行时又会根据描述是否为空选择不同分支，因此存在描述正确但效果走旧逻辑、结构化配置存在但未执行、效果生效但缺少可见反馈三类问题。

## 本轮 P0 修正

- 鸭：明确为“阵亡时，在原位置召唤 1 只鸭子”。死亡返场时在新鸭子身上显示召唤反馈，避免与普通兵营召唤混淆。
- 界面中的结构化召唤技能文案按触发器显示，不再把阵亡返场写成“召唤时额外召唤”。

## 全量检查规则

每张动物卡必须同时通过以下检查才可标记为已实装：

1. `skill_trigger` 有对应的运行时触发入口。
2. `skill_effect` 在该触发入口中有处理分支。
3. `skill_power` 与 `skill_cooldown_sec` 被实际读取，或文档明确说明固定值。
4. 卡牌详情描述与实际触发时机、目标、数值一致。
5. 玩家能在受影响动物身上看到图标与数值/状态反馈。

## 当前风险分组

- 结构化路径已存在：登场护盾/攻击/生命/速度/眩晕，攻击减速/眩晕/伤害/护盾/斩杀，受伤治疗/护盾/反伤，阵亡金币/召唤，占领金币，周期金币/治疗/护盾/修复。
- 需要继续专项验收：`aura` 光环的结构化数值一致性、`copy` 的统一定义、旧文案驱动的击杀/受伤/目标优先级技能，以及每个技能的冷却是否按配置执行。
- 无结构化技能的卡牌不等于无被动；其自然语言被动仍需逐张迁移后才能可靠验收。
